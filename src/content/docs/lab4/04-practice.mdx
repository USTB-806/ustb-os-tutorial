## 思考题（选做）

1. `find_pte_create` 中，为什么在分配新帧后要将 `FrameTracker` 存入
   `self.frames`？如果只写 `let frame = frame_alloc().unwrap();` 而不保存，
   会发生什么？在什么时候会出现错误？

2. SV39 中叶子 PTE 的 R/W/X 至少一位为 1，否则 CPU 无法区分叶子和中间节点。
   如果用户调用 `mmap(addr, len, 0)`（即 `prot = 0`，R/W/X 均为 0），查看
   `MapPermission::from_port` 的实现，思考内核会如何处理？

3. 当前的三级页表中，若某个中间节点下只有一个有效映射，`unmap` 删除该映射后，
   中间节点的页表帧并**不会**被立即释放（因为它在 `self.frames` 中）。这会导致
   什么问题？在什么场景下会出现？内核目前是如何处理的？

4. `translated_byte_buffer` 函数支持跨页的用户缓冲区（缓冲区可以跨越多个虚拟页）。
   阅读该函数的实现，说明它是如何将跨页的虚拟地址区间翻译为一个物理页切片列表的，
   并解释内核使用这种方式而非直接访问虚拟地址的原因。
